## Apa itu By-Example?

**Tutorial by-example** adalah jalur pembelajaran code-first yang dirancang untuk pengembang berpengalaman yang ingin belajar bahasa baru dengan cepat melalui contoh kode beranotasi lengkap.

**Target Audiens**: Programmer berpengalaman dan software engineer yang:

- Sudah menguasai setidaknya satu bahasa pemrograman dengan baik
- Ingin belajar TypeScript dengan cepat tanpa narasi panjang
- Lebih suka belajar melalui kode yang bisa langsung dijalankan
- Membutuhkan cakupan 95% secara efisien

## Cara Kerja Tutorial Ini

**Pendekatan Code-First**: Tampilkan kode terlebih dahulu, jalankan kedua, pahami melalui interaksi langsung.

**85 Contoh Beranotasi Lengkap**: Setiap contoh menggunakan notasi `// =>` untuk mendokumentasikan nilai, state, output, dan side effect di setiap baris penting.

**Struktur Lima Bagian** (per contoh):

1. **Penjelasan Singkat** (2-3 kalimat): Apa yang didemonstrasikan contoh ini
2. **Diagram Mermaid** (jika sesuai): Representasi visual konsep
3. **Kode Beranotasi Lengkap**: Contoh mandiri yang bisa langsung dijalankan
4. **Poin Kunci** (1-2 kalimat): Ringkasan wawasan inti
5. **Mengapa Penting** (50-100 kata): Relevansi produksi

**Contoh Mandiri**: Setiap contoh bisa langsung di-copy-paste dan dijalankan dalam cakupan bab-nya. Tidak ada dependensi eksternal atau referensi ke contoh sebelumnya.

**Cakupan 95%**: Tutorial ini mencakup 95% fitur TypeScript yang dibutuhkan untuk pekerjaan produksi melalui tiga tingkat kesulitan:

- **Pemula (Contoh 1-30)**: Cakupan 0-40% - Types, functions, interfaces, classes
- **Menengah (Contoh 31-60)**: Cakupan 40-75% - Generics, utility types, decorators, async
- **Lanjutan (Contoh 61-85)**: Cakupan 75-95% - Advanced types, compiler API, optimization

## Apa yang BUKAN By-Example

**BUKAN pengganti untuk**:

- Tutorial pemula (yang memberikan penjelasan mendalam untuk pemula lengkap)
- Quick Start (yang merupakan touchpoint cakupan 5-30%)
- Cookbook (yang berorientasi pada pemecahan masalah, bukan pembelajaran)

## TypeScript By-Example vs By-Concept

**Dua jalur pembelajaran yang saling melengkapi**:

| Jalur          | Pendekatan                               | Target Audiens           | Cakupan               |
| -------------- | ---------------------------------------- | ------------------------ | --------------------- |
| **By-Example** | Code-first, contoh beranotasi lengkap    | Pengembang berpengalaman | 95% melalui 85 contoh |
| **By-Concept** | Narratif-driven, penjelasan komprehensif | Pemula TypeScript        | 95% melalui deep dive |

**Pilih By-Example jika**:

- Anda menguasai JavaScript dengan baik dan ingin belajar TypeScript dengan cepat
- Anda lebih suka belajar melalui kode yang bisa langsung dijalankan
- Anda ingin cakupan komprehensif dengan cepat

**Pilih By-Concept jika**:

- Anda baru mengenal TypeScript atau sistem tipe
- Anda lebih suka penjelasan detail dan konteks
- Anda ingin pemahaman mendalam tentang keputusan desain

## Cara Menggunakan Tutorial Ini

**Baca Berurutan**: Contoh-contoh berkembang dari sintaks fundamental hingga penguasaan expert. Baca secara berurutan untuk hasil terbaik.

**Copy-Paste-Run**: Setiap contoh bersifat mandiri. Copy kode ke file `.ts`, compile dengan `tsc namafile.ts`, jalankan dengan `node namafile.js`.

**Fokus pada Anotasi**: Perhatikan komentar `// =>` yang menunjukkan nilai, state, dan output di setiap langkah.

**Pelajari Diagram**: Contoh kompleks dilengkapi diagram Mermaid yang memvisualisasikan aliran data, transisi state, atau layout memori.

**Konteks Produksi**: Bagian "Mengapa Penting" menghubungkan konsep dengan penggunaan dunia nyata di framework seperti React, Node.js, dan Express.

## Prasyarat

**Wajib**:

- Mahir dalam setidaknya satu bahasa pemrograman
- Pemahaman dasar JavaScript
- Node.js dan TypeScript terinstal (lihat [Initial Setup](/id/belajar/software-engineering/programming-languages/typescript/tutorials/initial-setup))

**Membantu tapi tidak wajib**:

- Pengalaman dengan pemrograman berorientasi objek
- Familiar dengan JavaScript modern (ES6+)
- Pemahaman tentang pemrograman asinkron
