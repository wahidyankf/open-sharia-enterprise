## Anti-Pattern Corrections

### Anti-Pattern 1: Common Mistake

**❌ Wrong Approach**:

```javascript
// Incorrect implementation
function wrongWay() {
  // This breaks because...
  return incorrectOperation();
}
```

**✅ Correct Approach**:

```javascript
// Correct implementation
function rightWay() {
  // This works because...
  return correctOperation();
}
```

**Why**: Explanation of what makes this wrong and how the correction fixes it.

## Comparison Examples

### Approach A vs Approach B

**Scenario**: Same goal, different approaches.

**Approach A** (Simple, fewer features):

```javascript
function simpleApproach() {
  // Straightforward implementation
  return basicResult();
}
```

**Approach B** (Complex, more features):

```javascript
function complexApproach(options) {
  // Feature-rich implementation
  const enhanced = addFeatures(options);
  return enhancedResult(enhanced);
}
```

**When to Choose**:

- Use Approach A when: Simple use case, performance critical
- Use Approach B when: Need flexibility, handling complex scenarios
