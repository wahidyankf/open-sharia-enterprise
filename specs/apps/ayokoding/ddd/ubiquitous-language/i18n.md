# Ubiquitous Language — i18n

**Bounded context**: `i18n`
**Maintainer**: ayokoding-web team
**Last reviewed**: 2026-05-10
**Audience:** Engineers, Technical Product/Project Managers

## One-line summary

Locale switching + tRPC i18n procedures + Next.js middleware. Application owns locale
negotiation and the middleware function; infrastructure owns translation file loaders;
presentation owns the locale switcher UI.

## Terms in detail

### Term: `Locale`

The two-letter ISO code identifying a content language: `"en"` (English, primary) or
`"id"` (Indonesian). All content paths and tRPC procedures that touch language scope take
a `Locale` parameter validated against this exact union. Owned by the i18n BC's application
layer.

**Code identifier(s)**:
`Locale` type and `localeSchema` Zod schema in
`apps/ayokoding-web/src/contexts/i18n/application/` (and re-exported from
`apps/ayokoding-web/src/lib/schemas/navigation.ts` for cross-BC consumption).

**Used in features**: `i18n/*.feature`

**Forbidden synonyms in this context**: "Language" (use `Locale` for the IDENTIFIER —
`"language"` is acceptable in human-prose UL discussion); "Lang code" (informal); "Region"
(locales here are language codes only, no region suffix).

**Related**: `Default locale`, `middleware.ts`, `Locale switcher`

---

### Term: `Default locale`

The `Locale` returned when no specific locale is requested or when locale negotiation fails:
`"en"`. Used by the `middleware.ts` to redirect bare-root requests (`/`) to `/en` and as
fallback when the request's `Accept-Language` matches no supported locale.

**Code identifier(s)**:
`DEFAULT_LOCALE` constant in
`apps/ayokoding-web/src/contexts/i18n/application/`.

**Used in features**: `i18n/*.feature`

**Forbidden synonyms in this context**: "Fallback locale" (use `Default locale`); "Primary
language" (use `Default locale`).

**Related**: `Locale`, `middleware.ts`

---

### Term: `middleware.ts`

The Next.js middleware function that runs on every request and performs locale negotiation:
redirects `/` to `/<Default locale>`, lets locale-prefixed paths (`/en/...`, `/id/...`)
pass through unchanged, and bypasses static assets and API routes. The implementation lives
inside the i18n BC's application layer; the conventional file `src/middleware.ts` is
reduced to a one-line re-export so Next.js still finds it at the conventional path.

**Code identifier(s)**:
`middleware` function (and `config` matcher) exported from
`apps/ayokoding-web/src/contexts/i18n/application/middleware.ts`.

**Used in features**: `i18n/*.feature`

**Forbidden synonyms in this context**: "Middleware" alone (use `middleware.ts` to point
at the file); "Locale router" (informal — use `middleware.ts` or describe the function).

**Related**: `Locale`, `Default locale`

---

### Term: `Locale switcher`

The presentation-layer React component in the `Header` that lets the user switch between
`Locale`s. Renders a dropdown of supported locales and navigates to the same `Slug` under
the new locale prefix on selection.

**Code identifier(s)**:
`LanguageSwitcher` component in
`apps/ayokoding-web/src/contexts/i18n/presentation/`.

**Used in features**: `i18n/i18n.feature`

**Forbidden synonyms in this context**: "Language picker" (the BC term is `Locale
switcher`); "Lang dropdown".

**Related**: `Locale`

---

### Term: `languages`

The tRPC procedure that returns the list of supported `Locale`s with their human-readable
labels (e.g. `[{code: "en", label: "English"}, {code: "id", label: "Bahasa Indonesia"}]`).
Consumed by the `Locale switcher` to populate its dropdown options.

**Code identifier(s)**:
`languages` procedure in
`apps/ayokoding-web/src/contexts/i18n/application/`.

**Used in features**: `i18n/i18n-api.feature`

**Forbidden synonyms in this context**: "Get languages" (use the procedure name);
"List locales".

**Related**: `Locale`, `Locale switcher`

---

### Term: `Translation`

A static lookup table mapping translation keys to localized strings, used in chrome-level
labels (header tooltips, locale switcher labels, error messages). Content body translation
is NOT done here — content lives as separate per-locale markdown files in the `content` BC.

**Code identifier(s)**:
Translation tables in
`apps/ayokoding-web/src/contexts/i18n/application/`.

**Used in features**: `i18n/*.feature`

**Forbidden synonyms in this context**: "Localized string" (informal); "Message" (too
generic).

**Related**: `Locale`

---

## Forbidden synonyms

- "Language" alone for the IDENTIFIER — use `Locale`.
- "Fallback locale" — use `Default locale`.
- "Language picker" — use `Locale switcher`.
