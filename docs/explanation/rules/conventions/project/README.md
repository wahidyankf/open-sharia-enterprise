# Project Conventions

Project-level documentation and planning organization standards.

## Purpose

These conventions define **HOW project planning and management documents should be organized**, covering the structure, naming, and lifecycle of planning documents in the `plans/` folder.

## Scope

**✅ Belongs Here:**

- Project planning document organization
- Plans folder structure and workflow
- Planning document naming standards
- Project-level documentation standards

**❌ Does NOT Belong:**

- Code/repository documentation (that's content/)
- Development workflow practices (that's development/)
- General file naming (that's meta/)
